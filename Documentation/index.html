Introduction

Hey there! Thank you for your interest in JupiterMeet Pro!

JupiterMeet Pro is a room-based audio/video conference solution as WebRTC SFU with a Signaling Server. It comes with a powerful admin panel written in Laravel.

It is a much more advanced version of JupiterMeet. Check out the difference here.

The system doesn't use any third-party services or APIs to host conferences. It is purely built upon WebRTC SFU. JupiterMeet Pro uses WebRTC technology so, the media will be end-to-end encrypted by default, which makes the system very secure.

JupiterMeet Pro is divided into two parts:

  1. A Web application in Laravel
  2. A Signaling server in NodeJS

Application Flow

Get started in 3 simple steps!

  1. Login
  2. Create a Meeting
  3. Join the Meeting

Optionally, you can send a meeting invite from the dashboard and copy the meeting link too. Other participants will be seen in the meeting when they join the same meeting via a link or Meeting ID.

By default, the Moderator rights feature is enabled. So, the other moderator/host can decide whom to allow into the meeting. The participants can be kicked out of the meeting by the moderator/host.

If the Auth mode feature is disabled, users are not required to create an account or meeting to start one. They can simply enter any Meeting ID and join the meeting.

The admin can change the logo, application name, theme color, etc from the portal itself. The meeting time is unlimited.

Users can enjoy the built-in features such as whiteboard, recording, chat, file sharing, screen sharing, picture-in-picture mode, full-screen mode, one-tap link sharing, etc.

JupiterMeet Pro has a fully responsive design and rich UI, which makes it very easy to use in mobile browsers too.

SaaS

The admin can create and set plans and their features in the SaaS mode. The end-user can then use Stripe and PayPal to subscribe to plans and use the features set by the admin.

Note: For iOS (V11+) mobile devices, only Safari browser is supported.

Requirements

The system works in any server that has PHP and NodeJS installed

  • VPS or dedicated server
  • At least 8 GB RAM
  • Web server & SSL
  • PHP >= 8.1
  • MySQL
  • NodeJS >= 16
  • Python 3.9

HTTPS is required

The application will run only on an HTTPS connection. Because HTTPS is required by the browser to get the media devices.

Notes:

  • The installation manual will help you install all the dependencies
  • Shared hosting will not work (unless you host the server separately)
